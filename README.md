## Database Schema

9 migrations with JWT-based RLS — every query scoped to `auth.jwt() -> 'user_metadata' ->> 'client_id'`.

| Table | Purpose |
|:------|:--------|
| `clients` | Tenant accounts — plan, status, industry, metadata |
| `agent_deployments` | Agent instances per client with config, status, metrics |
| `business_knowledge` | Centralized KB — FAQ, services, pricing, social, industry config |
| `customer_memory` | Long-term customer profiles — preferences, allergies, history, sentiment, events |
| `conversation_summaries` | AI-generated conversation index for context retrieval |
| `activity_logs` | Event stream powering dashboard analytics |
| `api_keys` | Client API authentication tokens |
| `calendar_configs` | Encrypted calendar provider credentials |
| `owner_channels` | Owner Brain WhatsApp channel configuration |

## Quick Start

```bash
# Clone and install
git clone <repo-url> && cd project-agent
pnpm install

# Dashboard (agents.dcp.sa)
pnpm --filter @project-agent/client-dashboard dev

# Website (marketing)
pnpm --filter @project-agent/website dev
```

### Required Environment Variables

```env
# Supabase
NEXT_PUBLIC_SUPABASE_URL=
NEXT_PUBLIC_SUPABASE_ANON_KEY=
SUPABASE_SERVICE_ROLE_KEY=

# AI
MINIMAX_API_KEY=                 # MiniMax M2.7 for all agent intelligence
FIRECRAWL_API_KEY=               # Live web search + context enrichment

# WhatsApp
WHATSAPP_BUSINESS_API_KEY=       # Multi-tenant WhatsApp provisioning

# Email
RESEND_API_KEY=                  # Transactional emails (agents@dcp.sa)

# Calendar (optional)
GOOGLE_CALENDAR_CLIENT_ID=
GOOGLE_CALENDAR_CLIENT_SECRET=
```
